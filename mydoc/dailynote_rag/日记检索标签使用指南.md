# 日记检索标签使用指南

本文档聚焦“日记检索语法中的标签/修饰符”，用于 `[[...]]` 与 `《《...》》` 日记本占位符的检索增强与行为切换。实现入口在 [RAGDiaryPlugin.js](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/RAGDiaryPlugin.js)。

## 一、当前支持的标签（修饰符）清单

以下修饰符通过 `::` 追加到日记本占位符中使用：

- `::Time`：时间感知检索  
  触发时间跨度解析 + 语义路/时间路双通道检索。
- `::Group`：语义组增强  
  触发语义组检测并增强查询向量。
- `::Rerank`：精排  
  先多召回，再进行重排筛选。
- `::TagMemo`（可带权重，如 `::TagMemo0.25`）  
  启用 TagMemo 标签增强与结构去重。
- `::AIMemo`  
  进入 AI 推理检索模式（受许可证控制）。

## 二、单独使用时的作用与适用场景

### 1) `::Time`
- **作用**：在指定时间范围内检索，采用“语义路 + 时间路”双通道，并对时间路结果按相关性排序。  
  [RAGDiaryPlugin.js:L1951-L2014](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/RAGDiaryPlugin.js#L1951-L2014)
- **适用场景**：  
  - 明确以时间为线索的问题，如“上周/去年/最近一次…”。  
  - 需要对一个时间窗口内的日记进行集中回溯。

### 2) `::Group`
- **作用**：检测语义组并增强查询向量，提高跨主题/相关主题的召回稳定性。  
  [RAGDiaryPlugin.js:L1921-L1927](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/RAGDiaryPlugin.js#L1921-L1927)
- **适用场景**：  
  - 主题跨度较大，但又存在“同组语义”关系。  
  - 你已在语义组中维护了一批概念，希望检索更贴近该组。

#### 2.1 语义组配置与参数

语义组配置文件位于：
- 主配置：`Plugin/RAGDiaryPlugin/semantic_groups.json`  
- 编辑配置：`Plugin/RAGDiaryPlugin/semantic_groups.edit.json`  
系统会优先读取并同步 `.edit.json` 到主配置文件。  
[SemanticGroupManager.js:L14-L51](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/SemanticGroupManager.js#L14-L51)

**配置结构示例**（来自示例文件）：
- `config`：包含 `min_cooccurrence_to_learn`、`min_cluster_size_for_new_group`、`auto_save_on_learn` 等字段。  
  这些字段在示例文件中存在，但当前版本未在运行时逻辑中直接使用，主要作为预留与记录。  
  [semantic_groups.json.example](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/semantic_groups.json.example)
- `groups.<组名>`：  
  - `words`：手工词元集合（核心词）。  
  - `auto_learned`：自动学习词元（如为空则不会参与匹配）。  
  - `weight`：组权重，用于放大或减弱该组的向量混合影响。  
  - `vector_id`：组向量缓存 ID（系统自动维护）。  
  - `last_activated` / `activation_count`：最近激活时间与激活次数（系统自动维护）。  
  [SemanticGroupManager.js:L273-L309](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/SemanticGroupManager.js#L273-L309)

#### 2.2 语义组生效机制

- **激活判定**：文本中命中组内词元（`words + auto_learned`）即可激活；  
  激活强度 = 匹配词数量 / 组词总数。  
  [SemanticGroupManager.js:L273-L295](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/SemanticGroupManager.js#L273-L295)
- **向量混合**：激活组会把“组向量”与“查询向量”加权融合，权重 = 组权重 × 激活强度。  
  [SemanticGroupManager.js:L414-L432](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/SemanticGroupManager.js#L414-L432)
- **组向量计算**：系统把“组名 + 词元列表”拼成描述文本生成 embedding，并缓存为组向量。  
  [SemanticGroupManager.js:L358-L379](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/SemanticGroupManager.js#L358-L379)

#### 2.3 语义组最佳实践

- **词元要“准而不泛”**：避免“很常见但无区分度”的词，如“好”“东西”，优先使用领域术语、缩写、专有名词。  
- **保持组内一致性**：同一语义组尽量围绕一个主题，不要把无关概念混进同组。  
- **权重用于“轻微引导”**：  
  - 权重大 → 更强的主题牵引，但可能过度聚焦；  
  - 权重小 → 更接近原始语义检索。  
  实际上更推荐先调词元，再调权重。  
- **通过管理面板或编辑插件维护**：  
  可在管理面板语义组编辑器或 `SemanticGroupEditor` 插件中修改，系统会同步并重新计算组向量。  
  [SemanticGroupEditor.js](file:///home/zh/projects/VCPToolBox/Plugin/SemanticGroupEditor/SemanticGroupEditor.js)

### 3) `::Rerank`
- **作用**：先扩大召回，再精排，提升“最相关结果”排序准确性。  
  [RAGDiaryPlugin.js:L1903-L1905](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/RAGDiaryPlugin.js#L1903-L1905)、  
  [RAGDiaryPlugin.js:L2083-L2094](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/RAGDiaryPlugin.js#L2083-L2094)
- **适用场景**：  
  - 关心“第一条/前几条”质量的查询。  
  - 日记数量大、容易出现排序噪声时。

### 4) `::TagMemo` / `::TagMemo0.25`
- **作用**：启用 TagMemo 增强向量与标签结构去重，提升“主题一致性”与“结构召回”。  
  [RAGDiaryPlugin.js:L1929-L1947](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/RAGDiaryPlugin.js#L1929-L1947)、  
  [KnowledgeBaseManager.js:L315-L343](file:///home/zh/projects/VCPToolBox/KnowledgeBaseManager.js#L315-L343)
- **适用场景**：  
  - 查询意图发散、容易跑题。  
  - 你有较高质量 Tag 体系，希望检索依赖标签结构。

### 5) `::AIMemo`
- **作用**：调用 AI 推理式检索流程（需许可证），非传统向量检索。  
  [RAGDiaryPlugin.js:L1127-L1169](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/RAGDiaryPlugin.js#L1127-L1169)
- **适用场景**：  
  - 需要“总结/推理/跨条目整合”的复杂回溯。  
  - 向量检索难以覆盖的隐含逻辑链。

## 三、混合使用的作用与推荐场景

### 常见组合与效果

- **`::Time + ::Rerank`**  
  时间范围内“先多召回再精排”，在时间窗口里挑最相关。  
  适合“在上个月里最关键的几条是什么？”

- **`::TagMemo + ::Rerank`**  
  先用标签增强锁定主题，再用精排打磨前几条。  
  适合“主题明确但结果排序容易漂”的场景。

- **`::Group + ::TagMemo`**  
  先用语义组拓宽主题边界，再用 TagMemo 拉回结构一致性。  
  适合“跨主题但仍需结构聚焦”的问题。

- **`::Time + ::TagMemo`**  
  时间范围内做标签增强，既限制时间又保证主题一致。  
  适合“最近一段时间某个主题的演变”。

- **`::AIMemo + ::Time`（仅在 AIMemo 授权且实现支持时）**  
  用 AI 推理在时间范围内做总结与归纳。  
  适合“用自然语言总结一段时间的进展”。

## 四、每种使用方式的最佳实践

### 1) 只用 `::Time`
- **最佳实践**：问题里明确包含时间线索，并尽量具体。  
  示例：`[[小吉日记本::Time]]` + “上周三之后我做了哪些优化？”

### 2) 只用 `::Group`
- **最佳实践**：提前维护好语义组，并确保组内概念一致。  
  示例：`[[小吉日记本::Group]]` + “回顾近期与向量检索相关的思考”

### 3) 只用 `::Rerank`
- **最佳实践**：当你更关心“前 3 条质量”，就开启精排。  
  示例：`[[小吉日记本::Rerank]]` + “最重要的三条总结是什么？”

### 4) 只用 `::TagMemo`
- **最佳实践**：Tag 体系完善时启用，效果最稳定。  
  示例：`[[小吉日记本::TagMemo0.25]]`  
  - 建议权重范围：0.10 ~ 0.35  
  - 权重过高可能过拟合标签，过低增强效果不明显。

### 5) 只用 `::AIMemo`
- **最佳实践**：用于“总结/推理/跨条目整合”，且对时效要求低。  
  示例：`[[小吉日记本::AIMemo]]` + “把这一年里我对日记系统的理解演进总结一下”

### 6) 组合策略
- **时间检索 + 精排**：`::Time + ::Rerank`  
  - 适合“时间内的最相关/最关键”问题。  
- **主题一致 + 精排**：`::TagMemo + ::Rerank`  
  - 适合“主题很明确但内容多”的情况。  
- **跨主题 + 结构聚焦**：`::Group + ::TagMemo`  
  - 适合“多主题联合检索但仍要结构化结果”的问题。

## 五、TagMemo 权重省略的含义

当你写 `::TagMemo` 但不显式给出权重（如 `::TagMemo0.25`），系统会使用**动态计算出的默认权重**来执行 TagMemo 增强。  

- 解析逻辑：如果没有数字，`tagWeight` 会采用 `defaultTagWeight` 参数；  
  如果包含数字，才使用显式权重。  
  [RAGDiaryPlugin.js:L1892-L1896](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/RAGDiaryPlugin.js#L1892-L1896)

- `defaultTagWeight` 的来源：  
  由 RAGDiaryPlugin 在检索流程中根据当前语义特征动态计算，随后传入 `_processRAGPlaceholder` 作为默认 TagMemo 权重。  
  [RAGDiaryPlugin.js:L1840-L1852](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/RAGDiaryPlugin.js#L1840-L1852)、  
  [RAGDiaryPlugin.js:L460-L483](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/RAGDiaryPlugin.js#L460-L483)

**实践建议**：  
- 不确定权重怎么配时，直接使用 `::TagMemo`，让系统自动给出合适权重。  
- 想要更强的主题收敛才显式加权重，且尽量从小值开始微调。  

## 六、语法提示（避免踩坑）

- **K 倍率**：用 `:1.2` 这种形式追加在修饰符串里，表示放大/缩小召回数量。  
  例如：`[[小吉日记本::TagMemo:1.5]]` 中的 `:1.5` 会被识别为 K 倍率。  
  实现：[_extractKMultiplier](file:///home/zh/projects/VCPToolBox/Plugin/RAGDiaryPlugin/RAGDiaryPlugin.js#L1543-L1546)
- **聚合语法**：用 `A|B|C` 表示多日记本聚合检索。  
  例如：`[[物理|政治|python::Rerank:1.2]]`
