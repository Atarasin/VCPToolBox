# AIMemo 一页说明：系统提示词模板 + 典型场景

本页聚焦“如何在系统提示词里正确启用 AIMemo，以及适合的典型使用场景”。内容基于当前代码实现与实际调用语法。

---

## 1) 系统提示词模板（复制即用）

### 模板 A：单日记本 AIMemo

```
[[AIMemo=True]]

你拥有以下回忆能力：
1) 需要“总结/推理/跨条目整合”时，使用 AIMemo。

请在需要时调用：
[[项目复盘日记本::AIMemo]]
```

### 模板 B：混合模式（先过阈值再 AIMemo）

```
[[AIMemo=True]]

当且仅当当前话题高度相关时，才进入深度回忆：
《《产品策略日记本::AIMemo》》
```

### 模板 C：跨库聚合 AIMemo

```
[[AIMemo=True]]

当需要跨库综合回忆时调用：
[[项目复盘|研发日志|客户反馈日记本::AIMemo]]
```

---

## 2) 语法要点（最容易踩坑的地方）

- `[[AIMemo=True]]` 必须写在 system 消息中，否则 `::AIMemo` 不生效
- `[[...::AIMemo]]` 直接进入 AIMemo，不走向量检索
- `《《...::AIMemo》》` 会先做相似度门控，达标才进入 AIMemo
- AIMemo 是“AI 推理式检索”，不支持 `::Time/::Group/::Rerank/::TagMemo` 的向量检索能力叠加

---

## 3) 典型场景（什么时候用 AIMemo）

### 场景 1：年度/季度复盘总结

```
[[年度复盘日记本::AIMemo]]
```

适合“归纳全年的经验演变、策略总结、关键决策复盘”。

### 场景 2：跨条目推理与因果追踪

```
[[项目复盘日记本::AIMemo]]
```

适合需要 AI 进行“因果链整理、阶段性复盘、影响分析”的场景。

### 场景 3：跨库联合回忆

```
[[产品|运营|研发日记本::AIMemo]]
```

适合需要横向对比与综合结论的场景。

### 场景 4：只在相关时回忆（控制污染）

```
《《战略方向日记本::AIMemo》》
```

适合日记内容体量大、不希望无关内容污染上下文的场景。

---

## 4) 输出特征（你会看到什么）

典型返回格式：

```
这是我获取的所有相关知识/记忆[[...]]
```

跨库聚合会带前缀：

```
[跨库联合检索: A + B + C]
这是我获取的所有相关知识/记忆[[...]]
```

---

## 5) 自动注入与调用控制

### 是否会自动注入

- 只要 system 提示词中同时包含 `[[AIMemo=True]]` 与 `[[项目复盘日记本::AIMemo]]`，AIMemo 会在预处理阶段自动执行，并把结果注入到 system 提示词中。

### 如何控制“在需要的时候调用”

1) 使用阈值门控  
```
《《项目复盘日记本::AIMemo》》
```
先做相似度门控，达标才进入 AIMemo。

2) 模板切换  
将 `[[...::AIMemo]]` 放入可切换的 system 模板，需要时启用，不需要时移除。

3) 变量开关  
用变量承载 AIMemo 语句，例如 `{{VarAIMemoBlock}}`，仅在需要时把变量值写成 `[[...::AIMemo]]`。

---

## 6) 推荐策略（一句话记住）

当你需要“总结 + 推理 + 跨条目整合”，用 `::AIMemo`；当你需要“精准片段召回”，用 `[[...]]` 或 `《《...》》` 的 RAG 模式。
