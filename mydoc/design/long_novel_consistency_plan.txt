长篇小说一致性RAG方案（基于VCP现有资源）

一、问题与目标
- 问题：日记内容简洁，跨卷世界观与大纲信息容易丢失，导致长篇小说前后不一致。
- 目标：把“世界观、故事大纲、时间线、角色设定、伏笔/反转、一致性校验”沉淀为可检索的结构化记忆，并在写作时按需召回，保证逻辑与设定连贯。

二、可复用资源（项目已有）
1) TagMemo 与 RAG 日记检索能力
- 动态K与聚合检索、Tag权重与阈值、Time/Group/Rerank/AIMemo等修饰符已经存在，支持跨日记本精准召回与时间线回溯。
  参考：[dailynote.md:L180-L336](file:///home/zh/projects/VCPToolBox/dailynote.md#L180-L336)
2) 专用创作 Agent 资产
- 大纲总编 SagaPlanner：提供卷级规划、一致性校验与变更记录模板。
  参考：[SagaPlanner.txt:L1-L217](file:///home/zh/projects/VCPToolBox/Agent/SagaPlanner.txt#L1-L217)
- 世界观 WorldForge：提供世界观结构化档案与一致性校验模板。
  参考：[WorldForge.txt:L1-L191](file:///home/zh/projects/VCPToolBox/Agent/WorldForge.txt#L1-L191)

三、总体方案概述
建立“分层记忆资产 + 结构化沉淀 + 检索策略 + 一致性校验闭环”的创作流程：
1) 结构化沉淀：把关键设定写入专门日记本，减少“碎片记忆”。
2) 分层召回：写作时按需组合多个日记本进行检索，确保世界观与大纲同时在场。
3) 一致性校验：每卷/关键情节变更后做结构化校验并回写。
4) 版本与变更记录：保证“改动可追踪、影响可评估”。

四、日记本体系设计（核心）
建议建立以下日记本（命名可按项目偏好调整）：
1) 世界观日记本
- 用 WorldForge 模板沉淀：世界观档案 + 变更记录 + 一致性校验。
2) 故事大纲日记本
- 用 SagaPlanner 模板沉淀：卷数决策 + 逐卷规划 + 跨卷衔接 + 自检清单 + 变更记录。
3) 时间线日记本
- 记录事件序列、时间跨度、跨卷间隔、旅行/通信成本等。
4) 角色设定日记本
- 记录角色弧线、核心动机、能力边界、关系变动。
5) 伏笔与反转日记本
- 记录伏笔编号、埋点位置、回收位置、误导路径与回收效果。
6) 一致性校验日记本
- 记录每轮校验发现问题、影响范围与修正建议。

五、结构化写入模板（精简版）
1) 世界观条目模板（来自 WorldForge 结构）
【世界观档案】
- 概览：主题母题 / 世界尺度 / 文明密度 / 核心冲突
- 关键维度：地理/历史/力量体系/政治/经济/文化
- 关键矛盾与悬念
【变更记录】新增/修改/废弃
【一致性校验】发现问题/影响范围/修正建议
Tag: 世界观名, 二级分类, 主题标签1, 主题标签2

2) 大纲条目模板（来自 SagaPlanner 结构）
【卷数决策】区间/N/依据/卷粒度/可改项
【逐卷规划】卷名/阶段问题/阶段答案/关键节点/锚点
【跨卷衔接总表】主线进度/伏笔回收/角色弧线里程碑
【自检清单汇总】
【变更记录】
Tag: 小说名, 二级分类, 主题标签1, 主题标签2

3) 时间线条目模板
【时间点/时间段】YYYY-MM-DD 或卷X-事件序列
【事件摘要】一句话
【因果链】起因→过程→结果
【影响范围】角色/地点/制度
Tag: 小说名, 时间线, 事件名, 相关角色

六、检索策略（写作/续写时的召回组合）
写作时同时引入世界观 + 大纲 + 时间线 + 角色 + 伏笔，多日记本聚合检索：
1) 聚合检索组合
《《世界观日记本|故事大纲日记本|时间线日记本|角色设定日记本|伏笔反转日记本:1.2::Group::TagMemo::Rerank》》
2) 时间回溯（剧情回顾/前文一致性）
[[时间线日记本::Time::TagMemo]]
3) 逻辑链补全（复杂跨章因果）
[[一致性校验日记本::TagMemo::Rerank]]
4) 大规模一致性复盘（关键转折或跨卷大改）
[[故事大纲日记本::AIMemo]]

说明：以上语法与能力均已有实现，可直接使用。
参考：[dailynote.md:L185-L336](file:///home/zh/projects/VCPToolBox/dailynote.md#L185-L336)

七、一致性校验闭环（写作流程）
1) 结构设定阶段
- 使用 WorldForge 生成世界观档案 → 写入世界观日记本。
2) 大纲阶段
- 使用 SagaPlanner 生成卷级规划 → 写入故事大纲日记本。
3) 章节/场景输出阶段
- 写作前：检索“世界观+大纲+时间线+角色+伏笔”聚合记忆。
4) 变更阶段
- 一旦出现设定改动或剧情反转，执行一致性校验：
  - 因果链/动机强度/时间线/世界规则/信息披露顺序
  - 发现问题后写入一致性校验日记本
  - 同步更新大纲与世界观日记本的变更记录

八、Tag 与语义组策略
1) 为每个日记本配置核心标签与阈值（rag_tags.json）
- 世界观：世界名/核心冲突/力量体系/地理
- 大纲：卷数/主线/反派/主题
- 时间线：时间跨度/关键事件
- 角色：主角/反派/关系/能力边界
2) 建议建立语义组（Group）
- “世界规则/代价/限制”组，保证检索时规则不被忽略
- “角色弧线/欲望/伤口/谎言”组，保证人物一致性

九、落地步骤（无代码改动）
1) 按“五类核心日记本”建立目录与命名规范
2) 用 WorldForge 与 SagaPlanner 模板生成V1世界观与卷级大纲
3) 为日记本配置 Tag 与阈值，并建立语义组
4) 写作时使用聚合检索模板进行上下文注入
5) 每次大改执行一致性校验并回写

十、预期收益
- 世界观/大纲/时间线统一检索入口，减少遗忘
- 版本化变更记录可追溯，避免“越改越乱”
- 语义组与TagMemo提升检索精度，减少不相关召回
- AIMemo用于大型重构场景，补全隐含因果链
