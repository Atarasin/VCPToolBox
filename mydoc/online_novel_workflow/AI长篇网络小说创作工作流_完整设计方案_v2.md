## 一、工作流概述

### 1.1 设计理念

本工作流采用**三阶段递进式**设计，通过严格的质量闭环机制确保从模糊想法到完整小说的全链路质量可控。

### 1.2 三阶段总览

| 阶段 | 名称 | 核心任务 | 输入 | 输出 |
|------|------|----------|------|------|
| 阶段0 | 需求初始化 | 用户与Agent对话明确需求 | 用户模糊想法 | 小说需求文档（Markdown） |
| 阶段1 | 前期设定产出 | 四层设定+挑刺者评审 | 需求文档 | 完整设定包 |
| 阶段2 | 章节创作产出 | 预检+创作-审核-回流循环+状态更新 | 设定包 | 章节正文+状态文档 |

### 1.3 核心设计原则

1. **质量优先**：每层设定必须经过挑刺者评审（≥85分）才能入库
2. **单一事实来源**：需求文档作为后续所有阶段的唯一依据
3. **可追溯性**：所有产出按规范路径存入Wiki
4. **迭代优化**：辩论循环和回流机制确保质量持续提升
5. **状态追踪**：章节创作后自动更新人物、势力、伏笔状态

### 2.2 文档类型说明

| 文档类型 | 所属阶段 | 作用 | 详细程度 |
|---------|---------|------|---------|
| **需求文档** | 阶段0 | 创作需求的总纲 | 宏观概述 |
| **世界观设定** | 阶段1-Layer1 | 世界的基础规则和环境 | 详细设定 |
| **人物设定** | 阶段1-Layer2 | 所有重要人物的详细档案 | 详细设定 |
| **小说大纲** | 阶段1-Layer3 | 全书整体结构和分卷规划 | 中观规划 |
| **分卷大纲** | 阶段1-Layer4 | 每一卷的详细剧情规划 | 微观规划 |
| **章节细纲** | 阶段2 | 单个章节的详细创作指导 | 最详细 |

**重要区别**：
- **分卷大纲**（Volume Outline）：规划**每一卷**的整体内容，是章节细纲的**指导文件**
- **章节细纲**（Chapter Outline）：规划**单个章节**的具体内容，是章节创作的**执行文件**

## 三、阶段0：需求初始化

### 3.1 阶段目标

通过用户与需求分析师Agent的对话，将模糊的创作想法转化为**自然语言格式的Markdown需求文档**。

### 3.2 需求分析师Agent

| 项目 | 说明 |
|------|------|
| **Agent名称** | RequirementAnalyst（需求分析师） |
| **核心职责** | 与用户对话，提炼和明确创作需求 |
| **产出物** | 需求文档（requirements.md，Markdown格式） |
| **关键能力** | 需求挖掘、信息提炼、自然语言输出 |

### 3.3 需求文档结构

每份需求文档必须包含YAML元信息头：

```markdown
---
doc_type: "requirements"
doc_id: "REQ_{project_name}_{version}"
title: "{小说名称}创作需求文档"
version: "1.0"
status: "draft"
created_at: "2026-03-21"
updated_at: "2026-03-21"
created_by: "RequirementAnalyst"
confirmed_by: ""
confirmed_at: ""
tags: ["需求文档", "阶段0"]
---

# {小说名称}创作需求文档

## 一、核心创意

### 1.1 一句话概括
...

### 1.2 主角核心目标
...

### 1.3 核心冲突
...

### 1.4 情感基调
...

## 二、题材与风格
...

## 三、世界观规模
...

## 四、篇幅规划
...

## 五、人物规划
...

## 六、特殊要求
...

## 七、附录
...
```

| 文件 | 路径 |
|------|------|
| 需求文档 | `wiki/00_requirements/requirements.md` |
| 确认记录 | `wiki/00_requirements/confirmation_record.md` |

## 四、阶段1：前期设定产出

### 4.1 四层设定流程

```
Layer 1: 世界观设定 ──▶ 挑刺者评审 ──▶ 入库(wiki/01_worldbuilding/)
     │
     ▼
Layer 2: 人物设定 ──▶ 挑刺者评审 ──▶ 入库(wiki/02_characters/)
     │  (主角/配角/反派，每人一个文档)
     ▼
Layer 3: 小说大纲 ──▶ 挑刺者评审 ──▶ 入库(wiki/03_plot/)
     │
     ▼
Layer 4: 分卷大纲 ──▶ 挑刺者评审 ──▶ 入库(wiki/03_plot/)
     │  (每卷一个文档，是章节细纲的指导文件)
```

### 4.2 各设定Agent

#### 4.2.1 世界观设定Agent

| 项目 | 说明 |
|------|------|
| **Agent名称** | WorldBuilder（世界观架构师） |
| **输入** | 需求文档 |
| **输出** | 多篇Markdown文档（每篇含YAML元信息） |
| **内容** | 世界基础、核心规则、势力设定、地点设定 |

#### 4.2.2 人物设定Agent

| 项目 | 说明 |
|------|------|
| **Agent名称** | CharacterDesigner（人物设计师） |
| **输入** | 需求文档 + 世界观文档 |
| **输出** | 多篇Markdown文档（每人一个文档，含YAML元信息） |
| **内容** | 主角、配角、反派、人物关系图 |

#### 4.2.3 小说大纲规划Agent

| 项目 | 说明 |
|------|------|
| **Agent名称** | StoryPlanner（大纲规划师） |
| **输入** | 需求文档 + 世界观 + 人物设定 |
| **输出** | novel_outline.md（含YAML元信息） |
| **内容** | 全书结构、分卷安排、伏笔规划、结局 |

#### 4.2.4 分卷大纲规划Agent

| 项目 | 说明 |
|------|------|
| **Agent名称** | VolumePlanner（分卷大纲规划师） |
| **输入** | 小说大纲 + 卷号 |
| **输出** | volume_{n}.md（每卷一个文档，含YAML元信息） |
| **内容** | 章节节点规划、节奏分析、人物出场、伏笔规划 |

### 4.3 挑刺者Agent

| 项目 | 说明 |
|------|------|
| **Agent名称** | Critic（挑刺者） |
| **核心职责** | 审核各类设定文档 |
| **产出物** | JSON审核报告 |
| **通过标准** | ≥85分，无CRITICAL问题，MAJOR问题≤2个 |

### 4.4 Wiki路径规范

| 设定类型 | 路径 |
|----------|------|
| 世界观基础 | `wiki/01_worldbuilding/world_basic.md` |
| 核心规则 | `wiki/01_worldbuilding/core_rules.md` |
| 势力设定 | `wiki/01_worldbuilding/factions/faction_{id}.md` |
| 地点设定 | `wiki/01_worldbuilding/locations/location_{id}.md` |
| 主角设定 | `wiki/02_characters/protagonists/{char_id}.md` |
| 配角设定 | `wiki/02_characters/supporting/{char_id}.md` |
| 反派设定 | `wiki/02_characters/antagonists/{char_id}.md` |

## 五、阶段2：章节创作产出

### 5.1 阶段流程

```
阶段1输入
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│  预检Agent（仅执行一次）                                      │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │大纲完整性检查│  │依赖满足性检查│  │上下文充分性检查│       │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│  ┌─────────────┐  ┌─────────────┐                          │
│  │人物状态检查 │  │伏笔状态检查 │                          │
│  └─────────────┘  └─────────────┘                          │
└─────────────────────────────────────────────────────────────┘
    │
    ├── 不通过 ──▶ 返回阶段1指定阶段
    │
    ▼ 通过
┌─────────────────────────────────────────────────────────────┐
│  创作-审核循环（最多3次迭代）                                 │
│                                                             │
│   细纲创作 ──▶ 正文创作 ──▶ 质量审核 ──▶ 回流决策            │
│                                             │               │
│                              不通过 ────────┘               │
│                              (根据问题级别回流)              │
│                              • 严重→返回细纲                 │
│                              • 重要→返回正文                 │
│                              • 轻微→通过并记录               │
│                              通过 ──▶ 状态更新               │
│                                       伏笔记录               │
│                                          │                  │
│                                          ▼                  │
│                                    存档交付                  │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 预检Agent

| 项目 | 说明 |
|------|------|
| **Agent名称** | PreChecker（预检员） |
| **核心职责** | 检查章节创作的前置条件 |
| **执行时机** | 仅从阶段1进入阶段2时执行一次 |

### 5.3 章节细纲规划Agent

| 项目 | 说明 |
|------|------|
| **Agent名称** | ChapterPlanner（章节细纲规划师） |
| **输入** | 分卷大纲 + 人物状态 + 伏笔状态 |
| **输出** | outline_v{volume}_c{chapter}.md（含YAML元信息） |
| **内容** | 场景列表、人物动作线、伏笔处理、状态变化预告 |

### 5.4 章节创作Agent

| 项目 | 说明 |
|------|------|
| **Agent名称** | ChapterWriter（章节创作者） |
| **输入** | 章节细纲 + 上下文包 |
| **输出** | chapter_v{volume}_c{chapter}.md（含YAML元信息） |
| **内容** | 章节正文 |

### 5.5 质量审核Agent

| 项目 | 说明 |
|------|------|
| **Agent名称** | QualityReviewer（质量审核员） |
| **审核维度** | 一致性、完整性、基础质量 |
| **输出** | JSON审核报告 |

### 5.6 状态更新机制（新增）

章节正文通过审核后，必须执行**状态更新**：

| 状态类型 | 更新内容 | 存储位置 |
|---------|---------|---------|
| **人物状态** | 位置、实力、心境、关系变化 | `wiki/05_state/character_state.md` |
| **势力状态** | 声望、实力、关系、领地变化 | `wiki/05_state/faction_state.md` |
| **世界状态** | 重大事件、规则变化 | `wiki/05_state/world_state.md` |
| **伏笔状态** | 已埋设、已回收、待回收 | `wiki/05_state/foreshadowing_state.md` |

### 5.7 Wiki路径规范

| 类型 | 路径 |
|------|------|
| 章节细纲 | `wiki/04_chapters/outline/outline_v{volume}_c{chapter}.md` |
| 章节正文 | `wiki/04_chapters/content/chapter_v{volume}_c{chapter}.md` |
| 章节摘要 | `wiki/04_chapters/summary/summary_v{volume}_c{chapter}.md` |
| 审核记录 | `wiki/04_chapters/review/review_v{volume}_c{chapter}.md` |
| 人物状态 | `wiki/05_state/character_state.md` |
| 势力状态 | `wiki/05_state/faction_state.md` |

## 九、修改记录

### v2.0 主要变更

1. **阶段0需求输出格式变更**
   - 从YAML格式改为Markdown自然语言格式
   - 更便于人类阅读和理解
   - 更灵活，不受固定字段限制

2. **明确分卷大纲与章节细纲的区别**
   - 分卷大纲：每卷一个文档，是章节细纲的指导文件
   - 章节细纲：每章一个文档，是章节创作的执行文件

3. **细化Wiki路径规范**
   - 人物按主角/配角/反派分类，每人一个文档
   - 势力、地点也按类型分类存储
   - 添加05_state目录用于状态追踪

4. **添加文档YAML元信息规范**
   - 每篇Wiki文档开头必须包含YAML元信息
   - 包含doc_type、doc_id、status等关键字段
   - 便于文档管理和追溯

5. **阶段2添加状态更新机制**
   - 章节正文通过审核后，自动更新人物状态
   - 更新势力状态、世界状态
   - 记录伏笔的埋设和回收情况
