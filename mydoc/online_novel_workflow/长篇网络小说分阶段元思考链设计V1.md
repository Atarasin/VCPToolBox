# 长篇网络小说分阶段元思考链设计

版本：V1.0  
参考来源：[VCP元思考整理](file:///home/zh/projects/VCPToolBox/mydoc/dailynote_rag/VCP元思考整理.md)、[VCP元思考_长篇小说创作详细案例](file:///home/zh/projects/VCPToolBox/mydoc/dailynote_rag/VCP元思考_长篇小说创作详细案例.md)

## 1. 设计目标
基于长篇网络小说创作的八个关键阶段，设计“短链、专用、可维护”的元思考链。每条链只服务单一阶段，避免跨阶段混用，保证检索聚焦与输出稳定。

## 2. 阶段列表
1) 构思策划  
2) 世界观设定  
3) 人物设计  
4) 情节大纲  
5) 章节细纲  
6) 正文创作  
7) 修订润色  
8) 发布运营

## 4. 分阶段链设计
### 4.1 构思策划链（novel_ideation）
**阶段特征**：定位赛道、锚定受众爽点、确立差异化卖点。  
**链结构**：  
1. 定位校准簇  
2. 受众画像簇  
3. 爽点体系簇  
4. 赛道差异簇  
**产出聚焦**：定位闭环、受众画像、卖点清单、风险规避。  
**使用方式**：
```
[[VCP元思考:novel_ideation::Group]]
```

### 4.2 世界观设定链（novel_worldbuild）
**阶段特征**：先底层规则，再表层扩展，保证自洽。  
**链结构**：  
1. 底层规则簇  
2. 尺度与资源簇  
3. 历史因果簇  
4. 权力结构簇  
**产出聚焦**：不可违背规则、尺度约束、历史主轴、权力格局。  
**使用方式**：
```
[[VCP元思考:novel_worldbuild::Group]]
```

### 4.3 人物设计链（novel_character）
**阶段特征**：主配反三类角色动机与弧线稳定，避免OOC。  
**链结构**：  
1. 动机弧线簇  
2. 能力边界簇  
3. 关系张力簇  
4. 角色风险簇  
**产出聚焦**：欲望与代价、能力边界、关系网与冲突点。  
**使用方式**：
```
[[VCP元思考:novel_character::Group]]
```

### 4.4 情节大纲链（novel_plot_outline）
**阶段特征**：总纲与分卷纲，伏笔与主线闭环。  
**链结构**：  
1. 主线结构簇  
2. 冲突升级簇  
3. 节拍节点簇  
4. 伏笔回收簇  
**产出聚焦**：阶段问题与答案、关键节点、伏笔台账。  
**使用方式**：
```
[[VCP元思考:novel_plot_outline::Group]]
```

### 4.5 章节细纲链（novel_chapter_outline）
**阶段特征**：单章目标明确、事件推进清晰、钩子可执行。  
**链结构**：  
1. 章节目标簇  
2. 事件推进簇  
3. 钩子落点簇  
**产出聚焦**：单章事件链、信息增量、结尾钩子。  
**使用方式**：
```
[[VCP元思考:novel_chapter_outline::Group]]
```

### 4.6 正文创作链（novel_drafting）
**阶段特征**：连载节奏、爽点兑现、情绪曲线。  
**链结构**：  
1. 节奏循环簇  
2. 爽点兑现簇  
3. 情绪波动簇  
4. 钩子密度簇  
**产出聚焦**：节奏循环建议、爽点间隔、情绪起伏与钩子配置。  
**使用方式**：
```
[[VCP元思考:novel_drafting::Group]]
```

### 4.7 修订润色链（novel_revision）
**阶段特征**：设定一致性、人物一致性、语言质量控制。  
**链结构**：  
1. 设定一致簇  
2. 人设一致簇  
3. 语言修复簇  
**产出聚焦**：冲突清单、修复建议、质量提升点。  
**使用方式**：
```
[[VCP元思考:novel_revision::Group]]
```

### 4.8 发布运营链（novel_operation）
**阶段特征**：读者反馈驱动迭代但不改主线，合规优先。  
**链结构**：  
1. 反馈聚类簇  
2. 迭代评估簇  
3. 合规风险簇  
**产出聚焦**：可调整项、不可调整项、运营策略与风险预警。  
**使用方式**：
```
[[VCP元思考:novel_operation::Group]]
```

## 6. 维护与扩展规则
1) 每条链最多4簇，优先保证检索稳定性。  
2) 每个簇只写一种思维职责，不跨阶段混用。  
3) 每个元逻辑模块保持独立、可复用、可直接注入。  
4) 若阶段输出不稳定，优先降低K序列而非增加簇数。  
5) 需要自动路由时再引入 Auto 模式，先保证链质量。
