—————— 长篇网络小说全流程工作流（VCP适配） ————————
本变量用于给各阶段Agent提供全局流程认知。执行任何阶段任务前，先对齐本流程，再执行本阶段职责。

【总原则】
1. 底层锁死，表层生长；主线不动，细节迭代。
2. 锁死项仅人工可写，AI全程只读；生长项可迭代但不得反向突破锁死项。
3. 先元思考再生成；先校验再定稿；写后即归档并可检索。
4. 局部限定内容（人物/场景/支线/道具/伏笔/临时规则）禁止入锁死库，必须按需生成并闭环退场。

【知识库与调用】
1. 知识库分层：锁死项核心日记本 + 生长项动态日记本。
2. 子库命名遵循：`{项目代号}.{层级}.{职能}`。
3. 调用策略：固定规则优先全量注入，动态任务优先按需RAG，复杂任务启用 `::Group` 与 `::Rerank`。
4. 定稿后回写对应子库，并保留版本记录与摘要。

【六阶段总览】
阶段1 定位锚定（锁死定位）  
- 输出：赛道分析、对标拆解、核心定位方案、定位合规报告。  
- 结果：锁定核心主题/价值底线/受众锚点/创作禁忌。  

阶段2 核心设定（锁死层定稿 + 生长框架搭建）  
- 输出：锁死层设定方案、设定自洽校验、生长项框架手册、冲突处理规则。  
- 结果：完成世界观/核心人设/力量体系底层规则与框架层搭建。  

阶段3 主线大纲（主线锁死 + 分卷框架）  
- 输出：全本总纲、主线闭环报告、前3卷分卷框架、开篇10章单章大纲。  
- 结果：锁死主线里程碑与核心伏笔回收计划。  

阶段4 正文连载（单章小闭环循环）  
- 流程：前置锚定 -> 正文生成 -> 四重校验 -> 迭代修改 -> 人工复核 -> 定稿入库 -> 下一章。  
- 结果：每章都完成合规、主线、大纲、局部边界四重校验后入库。  

阶段5 运营迭代（分卷大闭环循环）  
- 流程：数据舆情分析 -> 优化方案 -> 锁死项校验 -> 人工审核 -> 更新生长项与版本 -> 下一卷。  
- 结果：在不触碰锁死项前提下优化节奏、爽点、局部内容与后续大纲。  

阶段6 完结沉淀（闭环与IP化）  
- 输出：全本完结校验、结局闭环、番外与IP衍生材料、全本复盘报告。  
- 结果：主线闭环、伏笔全回收、方法论沉淀。  

【非线性运行规则】
1. 正常路径：1 -> 2 -> 3 -> 4(循环) -> 5(循环并回到3/4) -> 6。
2. 允许回退：仅可在生长项范围回退到阶段2/3/4；绝对禁止回退修改锁死项。
3. 触发条件：
- 单章校验不通过 -> 触发阶段4小闭环；
- 单卷完结 -> 触发阶段5大闭环；
- 数据下滑/反馈集中 -> 触发分卷内小范围迭代。

【全流程强制校验】
1. 锁死项合规校验：阶段1/2/3后、每章后、每次优化方案后。
2. 主线一致性校验：阶段3后、每卷后、全本完结后。
3. 局部边界校验：每章后、单卷后、每次优化方案后。
4. 伏笔回收校验：单卷后、全本后。
5. 内容合规校验：每章后、全本后。

【绝对红线】
1. 禁止给AI开放锁死项写权限。
2. 禁止提前全量设计局部限定内容。
3. 禁止局部内容越界影响主线或改写锁死项。
4. 禁止跳过校验环节。
5. 禁止为迎合反馈而改写锁死项。
6. 禁止无版本控制的生长项迭代。
7. 禁止单卷限定内容跨卷遗留。
