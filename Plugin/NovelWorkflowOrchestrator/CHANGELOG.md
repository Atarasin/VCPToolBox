## 0.4.0 - 2026-03-18

- 新增 Week4 集成测试：`INIT -> ... -> COMPLETED` happy path 全链路验证。
- 新增 Week4 集成测试：`CH_REVIEW -> CH_REFLOW -> CH_GENERATE` 回流路径验证。

- 补充联调级场景验证脚本，确认人工冻结后恢复与分发行为符合预期。
- 发布准备：manifest 版本提升至 `0.4.0`，通过全量测试与语法检查。

## 0.3.0 - 2026-03-18

- 新增质量门禁管理器，支持设定评分阈值与章节覆盖率/字数比/一致性门禁判定。
- 新增人工介入管理器，支持停滞触发、冻结唤醒、人工回复恢复与 `abort` 终止。
- 扩展 TickRunner 治理链路：接入质量路由、停滞检测、manual_review 生命周期与恢复逻辑。
- 扩展 StateStore：新增 counters、manual_review、quality_reports 的持久化读写能力。
- 接入双计数器更新策略：设定辩论轮次计数、章节迭代计数及超限转人工信号。
- 扩展配置模型：新增 Week3 门禁与人工介入配置项并同步 manifest/config 样例。
- 新增 Week3 单元与集成测试，覆盖质量路由、人工冻结恢复、计数器超限与停滞触发场景。

## 0.2.0 - 2026-03-18

- 新增顶层状态机与章节子状态路由，实现 `acted/waiting/blocked` 回执驱动推进。
- 新增 Agent 映射解析器，支持 `state + substate` 路由与 `SUPERVISOR` 升级兜底。
- 新增唤醒上下文组装与唤醒分发器，落盘 `wakeupTask`、幂等键、重试元数据与回执状态。
- 扩展 tick 主循环：支持 stdin 回执输入、状态推进、分发唤醒、checkpoint 审计闭环。
- 扩展存储层：新增 wakeup 读写、回执写回、项目维度唤醒查询能力。
- 新增第 2 周测试集，覆盖状态机路由、角色解析升级、分发与回执推进路径。

## 0.1.0 - 2026-03-18

- 新增 static 插件清单与入口，支持 cron 驱动 tick 执行。
- 新增 tick 主循环空跑模式，输出标准化 JSON 结果。
- 新增 StateStore 持久化框架，支持目录初始化、项目状态存储、checkpoint 写入与审计落盘。
- 新增基础文件锁与原子写入能力，降低并发写入冲突风险。
- 新增第 1 周对应单元测试与集成测试，覆盖 tick 主循环与持久化主路径。
