# FeishuBridge 版本变更日志

## 0.6.2 - 2026-03-15

### 自动回复二进制下发增强：支持本地路径
- `BINARY_REPLY` 新增 `attachment_path` 字段支持，可通过本地绝对路径读取附件并回传飞书。
- 二进制发送逻辑改为 `binary_base64` 与 `attachment_path` 二选一，兼容模型在不同场景下的输出能力。
- 自动回复提示词中补充 `attachment_path` 用法，明确与 `binary_base64` 的互斥关系。

### 测试覆盖
- 新增 `attachment_path` 解析测试。
- 新增按 `attachment_path` 发送图片测试。
- 更新提示词测试，校验 `attachment_path` 关键指令存在。

### 验证结果
- 回归测试全部通过（27/27）。

## 0.6.1 - 2026-03-15

### 自动回复提示词增强：显式告知二进制下发格式
- 在注入到模型的飞书环境提示中新增 `BINARY_REPLY` 结构块说明，明确图片/文件自动下发的输出格式。
- 提示中补充 `message_type`、`mime_type`、`file_name`、`binary_base64` 字段约定，降低模型输出格式偏差。

### 测试覆盖
- 更新环境提示词测试，校验提示中包含 `BINARY_REPLY` 与 `binary_base64` 关键指令。

### 验证结果
- 回归测试全部通过（25/25）。

## 0.6.0 - 2026-03-15

### 普通对话自动回复支持二进制下发
- 在普通对话回复链路中新增 `BINARY_REPLY` 结构化块解析，支持模型直接输出图片/文件下发指令。
- 自动回复阶段新增二进制发送流程，调用飞书图片/文件发送能力将内容直接回传给用户。
- 二进制块与工具调用块可以同时存在，文本、工具卡片、二进制消息按顺序回写飞书。
- 当二进制发送失败时会自动回写失败提示，避免用户侧无感失败。

### 测试覆盖
- 新增二进制回复块提取测试。
- 新增二进制回复块解析测试。
- 新增无效二进制回复块忽略测试。

### 验证结果
- 回归测试全部通过（新增后总计 25/25）。

## 0.5.0 - 2026-03-15

### 多模态上下文增强：图片与文档直入模型上下文
- 入站图片消息不再只生成文本提示，改为向 `/v1/chat/completions` 注入 `content[]` 多模态消息（`text + image_url(data URI)`）。
- 入站文件消息在尺寸可控范围内注入文档 `data URI` 到上下文，避免仅凭本地路径导致模型不可读。
- 新增 `FEISHU_MAX_INLINE_ATTACHMENT_BYTES` 配置，默认 `2097152`，用于限制附件内联到上下文的最大字节数，超限自动回退为文本提示。
- 附件下载链路补充 `Content-Type` 识别，保存附件时同步构建 `data URI` 以供多模态输入复用。

### 验证结果
- 回归测试全部通过（新增后总计 22/22）。

## 0.4.0 - 2026-03-15

### 架构重构：按单一职责拆分模块
- 将 `FeishuBridge.js` 拆分为入口编排层，保留生命周期、路由与消息主流程编排。
- 新增 `lib/bridgeState.js`，承载状态管理、会话持久化、Agent 映射、归档与通知能力。
- 新增 `lib/messageHelpers.js`，承载命令解析、消息归一化、工具块解析与格式构建能力。
- 新增 `lib/feishuGateway.js`，承载飞书 API 网关能力（上传、下载、推送、回写与聊天请求）。
- 保持原有对外导出契约与测试接口兼容，现有测试无需改动调用方式。

### 验证结果
- 重构后插件测试全部通过（19/19）。

## 0.3.0 - 2026-03-15

### Phase 2：二进制图片/文件上传与发送
- 推送接口 `POST /admin_api/feishu-bridge/push` 新增 `image` 与 `file` 消息类型。
- 新增二进制上传链路，支持 `binaryBase64` 输入并自动上传飞书资源后发送消息。
- 新增入站二进制消息解析，支持识别 `image_key`、`file_key`、`file_name`。
- 新增入站二进制资源下载与本地落盘，目录为 `state/attachments/<chat>/<date>/`。
- 入站图片/文件会被转换为结构化提示文本注入对话上下文。

### 测试覆盖
- 新增 `image/file` 推送类型解析测试。
- 新增入站图片消息归一化测试。
- 新增附件提示词构建测试。
- 新增图片与文件推送发送分支测试。
